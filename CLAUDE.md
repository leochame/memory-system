## 代码规范

### 架构红线（不可违反）

1. **四层架构是唯一基线**，不新增第五层
2. **保持 Memory-System 定位**，不偏离成通用 Agent 平台
3. **分层调用**：`cli -> memory/rag -> llm -> prompt`，禁止反向依赖
4. **统一写入**：记忆落盘必须经过 `MemoryWriteService`
5. **提取与通信解耦**：`LlmClient` 只通信，`LlmExtractionService` 做提取

### 代码风格

- Java 17+ 语法
- 使用 Lombok 注解（`@Slf4j`、`@Data`、`@RequiredArgsConstructor` 等）
- Spring Boot 依赖注入（构造器注入优先）
- 类职责单一，不做万能类
- 新增文件放在对应包下：`memory/`、`rag/`、`prompt/`、`skill/`、`task/`、`tool/`、`cli/`、`im/`、`llm/`

### 文件存储约定

- 运行时数据存放在 `.memory/` 目录
- Skill 文件：`.memory/skills/*.md`
- 向量存储：`.memory/vector_store.json`
- 用户画像：`.memory/user-insights.md`
- 元数据：`.memory/metadata.json`
