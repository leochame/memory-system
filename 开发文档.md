# 记忆管理系统开发文档

## 1. 文档定位

本文档只负责四件事：

1. 定义系统架构基线
2. 说明模块边界与依赖方向
3. 说明核心存储与主流程
4. 记录当前仍然有效的实现约束

不再承担路线图、内容规划或论文叙事职责；相关内容分别收敛到 `开发实现process.md`、`docs/开发计划.md`、`docs/记忆系统内容化方案.md`。

- 文档版本：v5.0
- 最后更新：2026-04-15

## 2. 项目概述

### 2.1 系统目标

Memory Box / Memory-System 是一个面向多轮对话的四层智能记忆系统，目标是让系统在长期交互中具备：

1. 短期上下文连续性
2. 用户画像沉淀能力
3. 历史经验复用能力
4. 记忆治理与证据解释能力
5. 自动评测与回归能力

### 2.2 技术栈

- Java 17+
- Spring Boot 3.2.0
- LangChain4j 0.35.0
- OpenAI 兼容接口
- all-MiniLM-L6-v2（本地嵌入）
- Jackson / Lombok / JLine

### 2.3 架构原则

1. 四层记忆架构是唯一基线，不新增第五层。
2. 分层调用方向固定为 `cli -> memory/rag -> llm -> prompt`。
3. 记忆写入统一经过 `MemoryWriteService`。
4. `LlmClient` 只负责模型通信与工具轮转，提取逻辑由 `LlmExtractionService` 承担。
5. 记忆载体统一落到文件系统，保持可查看、可恢复、可迁移。
6. 新能力优先在既有模块内增强，不破坏当前层次边界。

## 3. 四层记忆架构

### 3.1 总览

```text
Layer 1: Short-term Memory
Layer 2: Metadata
Layer 3: User Insight
Layer 4: Learned Knowledge
         ├── 4a Skill
         └── 4b Example / RAG
```

补充：

- `Agent.md` 是会话启动导航文件，不属于四层中的任何一层。
- 摘要、治理、评测、身份映射、主动服务都属于四层基线上的增强能力，而不是新层。

### 3.2 Layer 1 - Short-term Memory

职责：保存当前会话的即时上下文，支撑本轮连续性。

组成：

1. 最近完整对话窗口
2. 最近用户输入日志

当前实现：

- `conversation_history.jsonl`
- `recent_user_messages.jsonl`

主要读取点：

- `MemoryStorage.getRecentConversationTurns()`
- `MemoryStorage.getOlderUserMessages()`

### 3.3 Layer 2 - Metadata

职责：保存用户环境、助手偏好、全局控制和 CLI UI 设置。

当前实现：

- 存储文件：`metadata.json`
- 典型字段：
  - `user_interaction_metadata`
  - `assistant_preferences`
  - `global_controls`
  - `ui_settings`

主要入口：

- `MemoryStorage.readMetadata()/writeMetadata()`
- `CliRunner.showControls()/setControl()`

### 3.4 Layer 3 - User Insight

职责：保存用户长期偏好、背景和稳定习惯，是个性化回复的核心来源。

当前实现：

- 主正文：`.memory/user-insights.md`
- 冲突/待审核队列：`pending_explicit_memories.jsonl`
- 兼容结构：`memory_queues.json`
- 迁移源：`user_insights.json`（仅历史迁移）

写入来源：

1. 显式提取：用户消息中的直接偏好或事实
2. 隐式提取：夜间任务或 `/memory-update`

约束：

1. 用户画像以单 Markdown 文档为唯一正文入口。
2. 单条槽位式维护不再是主模式。
3. 兼容结构只用于运行时队列与历史迁移，不作为第二份画像正文。

主要入口：

- `MemoryExtractor`
- `LlmExtractionService.extractExplicitMemory()/extractUserInsights()`
- `MemoryWriteService`

### 3.5 Layer 4 - Learned Knowledge

职责：沉淀“怎么做事”的知识，而不是“用户是谁”。

#### 4a Skill

- 存储：`.memory/skills/*.md`
- 作用：沉淀方法论和稳定策略
- 加载方式：`load_skill(name)` 按需加载

#### 4b Example / RAG

- 存储：`vector_store.json`
- 作用：通过向量检索复用相似历史案例
- 当前能力：
  - 索引单条记忆
  - 重建向量索引
  - 语义检索
  - 构建对话期 RAG 上下文

## 4. 会话启动与 Prompt 组装

### 4.1 启动阶段

新会话启动时遵循以下规则：

1. 读取 `Agent.md` 作为导航上下文（若文件存在）
2. 在启用记忆时读取 `user-insights.md`
3. 不自动全量注入所有 Skill
4. 需要正文时再通过工具按需读取

### 4.2 Prompt 组装顺序

`SystemPromptBuilder` 当前组装顺序：

1. `Agent.md` 导航信息
2. Layer 2 Metadata
3. Layer 2 Assistant Preferences
4. Layer 1 较早用户消息窗口
5. Layer 3 用户画像正文
6. Layer 4a Skills（按需）
7. Layer 4b RAG Context / Examples

补充：

- 最近完整对话窗口通过 `messages` 单独传给模型。
- Prompt 负责组织多层记忆，不负责直接修改存储。

## 5. 核心增强能力

### 5.1 记忆反思（Memory Reflection）

作用：在回答前先判断当前问题是否需要长期记忆，以及需要哪类证据。

当前能力：

1. 判断是否需要记忆
2. 标记记忆使用目的
3. 记录证据类型与检索提示
4. 失败时可回退，不阻塞主链路

典型展示入口：

- `/memory-debug`
- `/memory-review`

### 5.2 证据链与解释视图

作用：说明“本轮回答为何参考了过去信息”。

当前能力：

1. 记录命中的 insight / example / skill / task
2. 区分 retrieved 与 used
3. 生成 evidence summary
4. 支持最近一轮或最近 N 轮调试查看

### 5.3 摘要与时间线

作用：在长会话中压缩上下文，并形成连续性证据。

当前实现文件：

- `session_summaries.jsonl`
- `topic_summaries.jsonl`
- `milestone_summaries.jsonl`

对应展示入口：

- `/memory-timeline`
- `/memory-scenes`
- `/memory-report`

### 5.4 记忆治理

作用：处理冲突、待审核和归档状态，提升系统可信度。

当前能力：

1. 显式记忆冲突入队
2. `/memory-governance` 查看治理状态
3. `/memory-approve <序号>` 批准写入
4. `/memory-reject <序号>` 拒绝并移出队列

### 5.5 主动服务与统一身份

作用：让记忆跨端共享，并参与任务提醒与周期性服务。

当前能力：

1. CLI / 飞书 / Telegram 多端接入
2. 统一身份映射
3. 定时任务创建与提醒
4. 主动提醒记录
5. 当前作用域周报

### 5.6 自动评测与 Benchmark

作用：为系统回归与实验提供最小自动评测闭环。

当前能力：

1. 单题 A/B 评测
2. 默认 benchmark 批量运行
3. 优先读取 `.memory/benchmark_questions.txt`
4. 写入 `eval_results.jsonl` 与 `benchmark_reports.jsonl`
5. CLI 查看题集、单题历史与批次汇总

## 8. 核心流程

### 8.1 对话主流程

1. 接收用户输入
2. 读取元数据、历史上下文、用户画像
3. 运行记忆反思
4. 按需组织 Skill / RAG / Task 等上下文
5. 调用模型生成回复
6. 异步写历史、提取显式记忆、更新访问计数

### 8.2 显式记忆流程

1. 用户输入进入主链路
2. `LlmExtractionService` 判断是否存在显式记忆
3. 如有冲突则入待审核队列
4. 通过 `MemoryWriteService` 写入或等待治理处理

### 8.3 隐式洞察流程

1. 定时任务或手动触发 `/memory-update`
2. 汇总近期对话与上下文
3. 提取稳定画像
4. 重写 `user-insights.md`

### 8.4 RAG 检索流程

1. 读取向量索引
2. 基于 query 做语义检索
3. 组织可注入上下文
4. 在回答阶段按需使用

### 8.5 Benchmark 流程

1. 优先读取外部题集
2. 逐题执行 A/B 评测
3. 写入单题结果与批次报告
4. 通过 CLI 查看历史与汇总
